# 🔥 Who's the super hot CPU??
> 개발 직군별 활동량을 기반으로 'CPU 온도'를 측정하여 랭킹을 매기고, 개발자 밈을 공유하고, 고민을 토로하는 개발자 전용 게이미피케이션 커뮤니티
## 1. 📖 프로젝트 개요 (Project Overview)
- **"Who's the super hot CPU??"** 는 개발자의 하루 활동(커밋 수, 커피 섭취량, 디버깅 횟수, 직군별 상이한 평가요소 등)을 입력받아 CPU 온도로 환산하고, 이를 통해 랭킹 경쟁, 칭호 획득, 그리고 직군별 소통을 즐길 수 있는 플랫폼입니다.
- 단순한 커뮤니티를 넘어, 역할(Role) 기반의 승급/강등 시스템을 통해 양질의 Q&A 문화를 형성하는 것을 목표로 합니다.
## 2. 👥 역할 및 승급 시스템 (Roles & Hierarchy)
> 본 서비스는 사용자의 활동과 평판에 따라 4가지 등급으로 나뉘며, 각 등급별로 권한과 의무가 엄격하게 관리됩니다.

| 역할 (Role) | 승급/강등 기준 | 주요 권한 및 특징 | 이용 가능 기능 |
| :--- | :--- | :--- | :--- |
| **Developer**<br>_(일반 유저)_ | • **기본 유저** (4개 직군 중 하나) | • 매일 CPU 온도 측정 대상<br>• **[의무]** 받은 답변 채택 여부 선택 필수 | • **고민:** 등록 O / 답변 X<br>• **밈:** 등록(하루 1회) / 좋아요 누르기(3개) |
| **Hot Developer**<br>_(일일 챔피언)_ | • **승급:** 전날 직군별 CPU 온도 1위<br>• **임기:** 24시간 (자정~자정) | • **[특권]** 익일 직군 특화 질문 2개 설정<br>• **[제약]** 당일 CPU 온도 측정 불가 | • **고민:** 등록 O / 답변 X<br>• **밈:** 등록(하루 1회) / 좋아요 누르기(3개) |
| **Optimizer**<br>_(고민 해결사)_ | • **승급:** Hot Developer 10회 + 칭호 5개<br>• **강등:** 답변 10회 이상 & 미채택률 80%↑ | • **[특권]** 고민 게시판 답변 전담<br>• **강등 시:** 누적 Hot Dev 횟수 초기화 | • **고민:** 등록 X / **답변 O**<br>• **밈:** 등록(하루 1회) / 좋아요 누르기 (3개) |
| **Root**<br>_(관리자)_ | • **승급:** 답변 50회 이상 & 채택률 80%↑ | • **[특권]** CPU 온도 가중치 로직 변경<br>• **[특권]** 밈 게시판 투표권 2배 (6개) | • **고민:** **등록 O / 답변 O**<br>• **밈:** 등록(하루 1회) / **좋아요 누르기 (6개)** |

## 3. 🏆 점수 산정 및 칭호 (Scoring & Badges)

매일 기록된 답변 값을 토대로 1등에게는 하루동안 특별한 **칭호(Badge)** 가 부여되며, 누적 칭호 획득 횟수 또한 승급에 영향을 줍니다.

### 🟢 공통 질문 (가중치 50%)
| 항목 | 칭호 (Badge Name) |
| :--- | :--- |
| 커밋 수 | 🤖 커밋 머신 |
| 마신 커피 (잔) | ☕ 내 몸의 70%는 아메리카노 |
| 수면 시간 | 😴 슬기로운 불면생활 |
| 개발 시간 | 💺 엉덩이가 무거워 |

### 🔵 직군별 질문 (가중치 30%)
| 직군 | 항목 | 칭호 (Badge Name) |
| :--- | :--- | :--- |
| **Frontend** | 페이지 구현 수 | 🎨 새 화면이 나를 부른다 |
| | API 연동 개수 | 📡 백-프론트 통역사 |
| | UI 변경으로 인한 코드 수정 | 🤯 "Figma 변경사항 확인해주세요"n번째 듣는 중 |
| | CSS/레이아웃 수정 횟수 | 🧩 CSS가 왜 그럴까 |
| **Backend** | API 설계/개발 개수 | 🛠️ JSON 상하차 중 |
| | 배포 여부 (0 or 1) | 🔥 Release 지옥에서 날 꺼내줘 |
| | 에러 로그 수집 건수 | 🚨 버그 담당 일진 |
| | DB 스키마 변경 건수 | 🛠️ ALTER TABLE 만능 노동자 |

| | 데이터셋 확보/정제 (GB) | 💀 라벨링 하다 영혼 가출 |
| | 실험 세팅 변경 횟수 | 💉 파라미터 튜닝 중독 |
| **Mobile** | 빌드 재시도 횟수 | 🔨 Gradle의 노예 |
| | 페이지 구현 수 | 🔄 컴포넌트 복붙 기계 |
| | SDK/Dependency 문제 해결 | 🔗 디펜던시 마스터 |
| | 크래시 발생 횟수 | 💔 앱은 죽었지만 난 살아있다 |

> **Note:** 나머지 가중치 20%는 **Hot Developer**가 매일 선정하는 '오늘의 특별 질문'에 부여됩니다.

## 6. 💻 주요 페이지 구성 (IA)

1.  **메인 페이지 (Dashboard)**
    * 오늘의 1등 유저 & 1등 직군 요약
    * 밈 게시판 베스트 & 고민 게시판 최신글
2.  **랭킹 페이지 (Ranking)**
    * 오늘의 랭킹 (전체 / 직군별) + TOP 3 카드 (카드에는 직군, 닉네임, 공통 질문에 대한 답변 값이 포함됩니다)
    * 누적 랭킹 (전체 / 직군별) + TOP 3 카드 (카드에는 직군, 닉네임, 공통 질문에 대한 답변의 *평균* 값)
3.  **고민 게시판 (Troubleshooting)**
    * **Developer/Hot Dev:** 질문 등록, 채택 (채택 전까지 점수 산정 Lock)
    * **Optimizer:** 답변 등록 (채택 시 점수 획득)
4.  **밈 게시판 (Meme)**
    * 하루 1회 등록 제한, 직군별/등급별 좋아요 투표권 차등 지급
